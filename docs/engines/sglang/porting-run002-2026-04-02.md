## Current Blocker

The active failure is now specific and reproducible:

- path: `radix_attention.py` -> `triton_backend.py` -> `memory_pool.py:set_kv_buffer()`
- failure type: `HSA_STATUS_ERROR_MEMORY_APERTURE_VIOLATION`
- kernel visible in the fault trace: `rotary_embedding_kernel`
- practical meaning: the ROCm radix + TurboQuant KV-cache write path is still invalid on this card in this fork

This is not evidence that AMD lacks radix support. It is evidence that the current ROCm port still has a broken KV write path under radix attention when `tq4` is exercised by a real generation request.
